Data & Sources

Field Name: Fire_Size_Class

Field Alias: Fire Size Class

Purpose: Classifies fire size into standard A to G categories using GIS acreage.

Description: Used to symbolize fire size and compare wildfire magnitude across the state.

Expression

classify(!GIS_ACRES!) 
def classify(acres):
    if acres is None:
        return None
    elif acres <= 0.25:
        return "A"
    elif acres < 10:
        return "B"
    elif acres < 100:
        return "C"
    elif acres < 300:
        return "D"
    elif acres < 1000:
        return "E"
    elif acres < 5000:
        return "F"
    else:
        return "G"

Field Name: Fire_Duration_Days

Field Alias: Fire Duration Days

Purpose: Calculates fire duration in days using alarm and containment dates.

Description: Supports duration analysis while filtering out invalid or inconsistent records.

Expression

duration_days(!ALARM_DATE!, !CONT_DATE!, !YEAR_!) 
from datetime import datetime

def duration_days(alarm_date, cont_date, fire_year):
    if alarm_date is None or cont_date is None or fire_year is None:
        return None

    if alarm_date.year < 1900 or cont_date.year < 1900:
        return None

    if abs(alarm_date.year - fire_year) > 1:
        return None
    if cont_date.year < fire_year - 1 or cont_date.year > fire_year + 2:
        return None

    a = datetime(alarm_date.year, alarm_date.month, alarm_date.day)
    c = datetime(cont_date.year, cont_date.month, cont_date.day)

    diff = (c - a).days

    if diff < 0:
        return None

    return diff

Field Name: Cause_Group

Field Alias: Cause Group

Purpose: Groups fire causes into Natural, Human-Related, and Unknown categories.

Description: Used for simplified thematic filtering and comparative cause analysis.

Expression

cause_group(!CAUSE!) 
def cause_group(cause):
    if cause is None:
        return "Unknown"
    elif cause in [1, 17]:
        return "Natural"
    elif cause == 14:
        return "Unknown"
    else:
        return "Human-Related"

Field Name: Discovery_Month

Field Alias: Discovery Month

Purpose: Extracts the month in which the fire was discovered.

Description: Supports month-based temporal interpretation and filtering.

Expression

get_month(!ALARM_DATE!) 
def get_month(alarm_date):
    if alarm_date is None:
        return None

    months = {
        1: "January", 2: "February", 3: "March", 4: "April",
        5: "May", 6: "June", 7: "July", 8: "August",
        9: "September", 10: "October", 11: "November", 12: "December"
    }

    return months.get(alarm_date.month, None)

Field Name: Fire_Season

Field Alias: Fire Season

Purpose: Groups fire records into seasonal categories.

Description: Used for season-based wildfire analysis and thematic filtering.

Expression

get_season(!ALARM_DATE!) 
def get_season(alarm_date):
    if alarm_date is None:
        return None

    month = alarm_date.month

    if month in [12, 1, 2]:
        return "Winter"
    elif month in [3, 4, 5]:
        return "Spring"
    elif month in [6, 7, 8]:
        return "Summer"
    elif month in [9, 10, 11]:
        return "Fall"
    else:
        return None

Field Name: Complex_Fire_Flag

Field Alias: Complex Fire Flag

Purpose: Identifies whether a fire is part of a complex fire incident based on COMPLEX_ID.

Description: Adds operational context and supports comparison between complex and non-complex incidents.

Expression

complex_flag(!COMPLEX_ID!) 
def complex_flag(complex_id):
    if complex_id is None:
        return "Not Complex"

    if str(complex_id).strip() == "":
        return "Not Complex"

    return "Complex"

Popup Configuration

Popup Styling & Content

Purpose: Configure wildfire popups so they are readable, structured, and consistent with the web app theme.

Where used: ArcGIS Online Web Map popup configuration

Components: Arcade expression for logic and HTML for presentation

Description: Arcade was used to organize wildfire attributes into a clean summary, while custom HTML was used to style the popup content with a darker background, padding, and rounded edges to better match the application interface.

Arcade Expression

// Arcade expression for popup: Detailed wildfire info in readable format

var fireYear = $feature["YEAR_"];
var fireClass = $feature["Fire_Size_Class"];
var fireAcres = $feature["GIS_ACRES"];
var fireDuration = $feature["Duration_Category"];
var fireCause = $feature["Cause_Group"];
var fireSeason = $feature["Fire_Season"];
var fireComplexity = $feature["Complex_Fire_Flag"];

// Handle null values
fireYear = IIf(IsEmpty(fireYear), "Not Available", Text(fireYear));
fireClass = IIf(IsEmpty(fireClass), "Not Available", fireClass);
fireDuration = IIf(IsEmpty(fireDuration), "Not Available", fireDuration);
fireCause = IIf(IsEmpty(fireCause), "Not Available", fireCause);
fireSeason = IIf(IsEmpty(fireSeason), "Not Available", fireSeason);
fireComplexity = IIf(IsEmpty(fireComplexity), "Not Available", fireComplexity);

// Burn area formatting
var burnArea = "Not Available";
if (!IsEmpty(fireAcres)) {
    burnArea = Text(Round(fireAcres, 0), "#,###") + " acres";
}

// Convert size class to readable label
var sizeLabel = Decode(
    fireClass,
    "A", "Very Small",
    "B", "Small",
    "C", "Moderate",
    "D", "Medium",
    "E", "Large",
    "F", "Very Large",
    "G", "Extreme",
    fireClass
);

// Slightly cleaner cause text
var causeLabel = fireCause;
if (fireCause == "Unknown") {
    causeLabel = "Unknown (No data reported)";
}

// Build popup text
var text =
    "📅 Year: " + fireYear + TextFormatting.NewLine +
    "📏 Size: " + sizeLabel + TextFormatting.NewLine +
    "🔥 Burn Area: " + burnArea + TextFormatting.NewLine +
    "⏱ Duration: " + fireDuration + TextFormatting.NewLine +
    "⚡ Cause: " + causeLabel + TextFormatting.NewLine +
    "🌦 Season: " + fireSeason + TextFormatting.NewLine +
    "🔗 Complexity: " + fireComplexity;

return text;

Interpretation

Discussion

The wildfire explorer supports multi-dimensional investigation of wildfire behavior across California by combining thematic classification, interactive filtering, and guided map exploration.

Patterns in Fire Size

Fire size classification reveals strong spatial variation across California, with larger fire classes concentrated in specific high-risk regions and smaller incidents appearing more broadly across the state.

Cause-Based Interpretation

Grouping causes into natural, human-related, and unknown categories helps simplify the original source values and supports clearer comparison of ignition patterns.

Temporal Insights

Discovery month, seasonal categories, and duration fields provide an accessible way to examine wildfire timing without relying only on raw date fields.

App Design Implications

The interactive app was intentionally designed around thematic layer exploration so users can investigate one analytical question at a time instead of being overwhelmed by all variables at once.

Data Constraints

Some source records contain incomplete or inconsistent temporal information, which required additional validation before duration-based fields could be used reliably.

Why This Matters

Together, these layers and filters make it easier to understand not only where wildfires occur, but how they differ in size, timing, cause, and operational context.
